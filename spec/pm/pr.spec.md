## 1. 产品环境（Product Environment）

<product-environment>

**产品名称：** 问凯利（Ask Kelly）

**标语：** 和凯利对话，看见自己建构的世界

**版本：** 2.0 → 目标 3.0

**产品描述：**

问凯利是一个以心理学家乔治·凯利（George Kelly）及其个人建构心理学为核心的**知识对话环境**。访问者可以在这里：
- 阅读关于凯利理论的深度文章
- 浏览系统化的个人建构心理学知识库
- **与凯利 AI 直接对话**，在对话中反思自己的构念系统

这个环境的核心动作空间是：**对话中看见**——通过与凯利的对话，访问者得以观察到自己的构念系统如何运作，从而获得新的行动可能。

**主要智能体：**

| 编号 | 智能体 | 动作能力 | 感知通道 |
|------|--------|----------|----------|
| A1 | 作者/内容创作者 | 撰写文章、更新知识库、配置部署 | 文件系统、Git、Vercel 控制台 |
| A2 | 访问者/读者 | 阅读、提问、浏览、探索 | 视觉界面（浏览器） |
| A3 | 凯利 AI | 回答、引导反思、阐释理论 | API 请求（System Prompt + 知识库上下文） |

**核心可供性（Core Affordances）：**

1. **对话可供性**（Primary）— 访问者可以向凯利提问，获得基于个人建构心理学的回应
2. **阅读可供性**（Secondary）— 访问者可以浏览凯利相关文章和知识库
3. **反思可供性**（Latent）— 通过持续的对话序列，访问者逐渐察觉自身的构念模式

</product-environment>

### 2.1 主要可供性 — 立即感知（Primary Affordances）

#### AFF-01：与凯利对话

| 属性 | 说明 |
|------|------|
| **动作** | 用户输入文字提问，凯利以个人建构心理学框架回应 |
| **层级** | Primary |
| **感知** | 对话输入框 + 发送按钮（视觉）；chat API endpoint（语义） |
| **对应实体** | E2（对话记录）、A3（凯利 AI） |
| **关联约束** | C1（AI 人格一致性）、C6（上下文长度管理） |

**环境属性：**

| 属性 | 值 | 可被感知 |
|------|-----|---------|
| 输入框 | `<textarea>` 或 `<input>` 元素 | 人类：视觉；AI：DOM 语义 |
| 发送按钮 | 带"发送"图标的 `<button>` | 人类：视觉；AI：role="button" |
| 对话历史 | 滚动消息列表 | 人类：视觉；AI：DOM 结构 |

**感知-动作耦合：**

| 通道 | 感知 | 动作 | 反馈 |
|------|------|------|------|
| 人类 | 看到输入框和发送按钮 | 输入文字并点击发送 | 消息出现在对话列表中 |
| AI | 检测到 chat API 调用 | 发送 POST 请求 | 收到流式响应（SSE） |

**依赖：**
- 需要：AFF-04（对话页面导航）
- 启用：AFF-03（反思）、AFF-05（构念洞察）

## 3. 环境约束（Environmental Constraints）

### 3.1 物理约束

| 约束 | 对可供性的影响 |
|------|--------------|
| 屏幕尺寸（移动端 vs 桌面端） | 对话输入框和文章列表需要响应式适配 |
| 输入方式（触摸 vs 键盘） | 对话输入框需同时支持触摸键盘和物理键盘 |
| 网络延迟 | 凯利 AI 回复需要流式输出（SSE），避免用户等待焦虑 |

### 3.2 结构约束

| 约束 | 对可供性的影响 |
|------|--------------|
| 当前无数据库 | 对话记录仅在会话内存中，刷新后丢失（AFF-05 反思可供性受限） |
| 静态页面架构 | 文章和知识库为预渲染内容，更新需手动操作 |
| 单一 API 端点 | 所有对话均通过 chat API 处理，无分流机制 |

### 3.3 安全约束（来自 real.md）

| 约束 ID | 约束 | 可供性限制 |
|---------|------|-----------|
| C2 | API 密钥安全 | AI 对话功能依赖服务端密钥，前端不暴露 |
| C4 | 对话数据隐私 | AFF-05（构念觉察）不能未经用户同意存储对话 |
| C7 | 代码托管安全 | .env 等配置文件不进入版本控制 |

## 5. 反馈机制（Feedback Mechanisms）

### 即时反馈（< 100ms）

| 动作 | 人类反馈 | AI 反馈 |

## 7. 非可供性（Explicitly Prevented Actions）

| 动作 | 被阻止的原因 | 说明 |
|------|-------------|------|
| 用户注册/登录 | 保持低门槛访问 | 当前版本不需要用户账号即可使用核心功能。若将来需要存储对话历史，可引入轻量认证 |
| 多人社交互动 | 聚焦"人与智者"对话 | 不引入评论、点赞等社交功能，保持对话的私密性和专注度 |
| 凯利之外的其他智者 | 当前聚焦凯利一人 | 后续版本（v4.0）再考虑扩展多智者 |
| 文件上传 | 保持界面简洁 | 对话仅支持文字输入 |
| AI 生成知识库内容 | 确保学术准确性 | 知识库内容由作者手动整理，确保基于真实文献 |
| 用户修改凯利人格设定 | 保持对话一致性 | 用户与真实的"凯利"对话，而非自定义角色 |
