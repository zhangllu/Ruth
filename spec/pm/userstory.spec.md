## 按 Complex Story 分组

### CS-01：对话凯利

**描述：** 访问者进入对话界面，向凯利提问，获得基于个人建构心理学的回应
**用户类型：** U1 心理学爱好者 / U2 教育工作者 / U3 个人成长实践者
**核心价值：** 与智者直接对话，获得个性化的理论回应
**对应可供性：** AFF-01（对话）、AFF-04（导航）
**故事类型分布：** Light 2, Dark 1, Grey 1

**用户旅程：**
看到凯利介绍 → 进入对话页面 → 提出问题 → 收到凯利回复 → 继续对话探索

---

#### MS-L-01：发起首次对话

**Complex Story：** CS-01
**故事类型：** Light（从不会到会）
**优先级：** P0
**故事点数：** 2

##### 用户故事

作为一个**心理学爱好者（U1）**，
我想**打开对话页面就能直接向凯利提问**，
以便**快速体验与凯利对话的感觉，不需要任何注册或配置**。

##### 故事评估（Light Story — 从坏到好）

- **起始状态**：用户听说过凯利但无法直接与他交流，只能阅读二手资料
- **转折点**：系统提供一个直接的对话入口，用户输入文字即可与凯利对话
- **结束状态**：用户发出了第一个问题，正在等待凯利的回应，感到期待和好奇
- **情绪体验**：从"想知道但无从问起" → "可以直接对话了，很神奇"

##### 验收标准

- [ ] 用户打开对话页面后，立刻看到输入框，无需登录或注册
- [ ] 输入框中有 placeholder 文字提示（如"请向凯利提问…"）
- [ ] 用户输入文字后，点击发送按钮或按回车即可发送
- [ ] 发送后，用户的问题出现在对话列表中

##### 用户可感知的变化

- **操作前**：看到空的对话界面 + 输入框
- **操作中**：输入文字，点击发送
- **操作后**：自己的问题出现在对话列表中，等待凯利回复

##### 约束

- C2：所有对话请求通过服务端 API 转发，前端不暴露密钥
- C6：首次对话无上下文压力，无需额外处理

##### 依赖

- **需要**：CS-01 对话页面导航可用
- **启用**：MS-L-02、MS-G-01

#### MS-L-02：接收凯利回复

**Complex Story：** CS-01
**故事类型：** Light（从不会到会）
**优先级：** P0
**故事点数：** 3

##### 用户故事

作为一个**个人成长实践者（U3）**，
我想**看到凯利以他特有的风格回应我的问题**，
以便**获得基于个人建构心理学的专业且有启发性的回答，而不仅仅是通用 AI 回复**。

##### 故事评估（Light Story — 从坏到好）

- **起始状态**：用户发送问题后，面对空白等待区，不确定能否得到有价值的回复
- **转折点**：凯利的回复以流式方式逐字出现，且风格鲜明（苏格拉底式引导、构念分析）
- **结束状态**：用户收到完整的凯利式回复，感到被理解和被启发
- **情绪体验**：从"不确定" → "惊喜，确实像在和心理学家对话"

##### 验收标准

- [ ] 凯利回复以流式方式逐字显示（类似打字效果）
- [ ] 回复内容基于个人建构心理学框架，不偏离凯利人格（约束 C1）
- [ ] 回复支持 Markdown 渲染（粗体、列表、引用等）
- [ ] 回复完成后，输入框重新可用，用户可继续对话
- [ ] 如果 API 调用失败，显示友好的错误提示

##### 用户可感知的变化

- **操作前**：看到"凯利正在输入…"或打字动画
- **操作中**：回复内容逐字出现，可开始阅读
- **操作后**：完整回复展示在对话中，输入框重新激活

##### 约束

- C1：回复必须基于凯利人格和理论框架
- C6：单次回复在 token 限制内完成

##### 依赖

- **需要**：MS-L-01（对话功能可用）
- **启用**：MS-G-01（持续对话）

#### MS-D-01：处理对话异常

**Complex Story：** CS-01
**故事类型：** Dark（从正常到异常）
**优先级：** P0
**故事点数：** 1

##### 用户故事

作为一个**心理学爱好者（U1）**，
我想**在对话出错时获得清晰的提示和恢复路径**，
以便**不会因为一次技术问题而丢失对话上下文或感到困惑**。

##### 故事评估（Dark Story — 从好到坏）

- **起始状态**：用户正顺利与凯利对话，一切正常
- **转折点**：网络中断、API 超时或返回异常，对话卡住
- **结束状态**：系统显示友好的错误提示，用户可选择重试
- **情绪体验**：从"顺畅交流" → "突然卡住" → "知道怎么回事，可以重试"

##### 验收标准

- [ ] API 超时或网络错误时，页面不崩溃，显示错误提示
- [ ] 错误提示包含人性化文案（如"凯利似乎在思考…请稍后再试"），而非技术报错
- [ ] 用户可点击"重试"按钮重新发送上一条消息
- [ ] 重试后对话上下文不丢失

##### 用户可感知的变化

- **操作前**：对话正常进行
- **操作中**：回复中断，出现提示信息
- **操作后**：用户了解状况并决定重试

##### 约束

- C1：错误提示文案符合凯利产品的整体风格，不突兀

##### 依赖

- **需要**：MS-L-02
- **启用**：无

#### MS-G-01：持续对话探索

**Complex Story：** CS-01
**故事类型：** Grey（循环往复）
**优先级：** P1
**故事点数：** 3

##### 用户故事

作为一个**个人成长实践者（U3）**，
我想**在对话中查看历史消息，并继续追问凯利**，
以便**逐步深入地探索某个话题，形成完整的对话线索**。

##### 故事评估（Grey Story — 循环往复）

- **循环模式**：提问 → 收到回复 → 理解 → 再提问 → 再收到回复
- **优化点**：对话历史可滚动查看，新回复自动滚动到底部
- **习惯形成**：用户从"试一下"到"习惯遇到问题时来问凯利"
- **情绪体验**：从"刻意使用" → "自然地持续对话"

##### 验收标准

- [ ] 对话列表可滚动查看历史消息
- [ ] 新消息自动滚动到底部（如果用户未手动滚动）
- [ ] 如果用户滚动查看历史，新消息不强制跳转（提供"回到最新"按钮）
- [ ] 对话上下文在单次会话中保持连续（凯利记得上一条对话内容）
- [ ] 输入框在对话过程中始终保持可见和可用

##### 用户可感知的变化

- **操作前**：看到完整的对话历史
- **操作中**：滚动查看旧消息，在底部输入新问题
- **操作后**：新回复出现在对话底部，历史消息依然可查阅

##### 约束

- C6：对话上下文长度需管理，超出 token 限制时需截断或压缩历史
- C4：对话记录仅在当前会话中保留，刷新后丢失

##### 依赖

- **需要**：MS-L-01、MS-L-02
- **启用**：MS-L-03、MS-L-04

**故事类型：** Light（从不知道到知道）
**优先级：** P0
**故事点数：** 2

作为一个**心理学爱好者（U1）**，

以便**快速了解有哪些内容可读，找到自己感兴趣的话题**。

- **操作后**：找到感兴趣的文章，点击进入阅读

#### MS-L-05：识别自己的构念

**Complex Story：** CS-03
**故事类型：** Light（从迷茫到清晰）
**优先级：** P1
**故事点数：** 5

##### 用户故事

作为一个**个人成长实践者（U3）**，
我想**在对话中，凯利帮助我识别出我自己的构念模式**，
以便**意识到我用来理解世界的隐形框架，进而获得新的选择可能**。

##### 故事评估（Light Story — 从坏到好）

- **起始状态**：用户感到困惑或卡住，但不知道为什么
- **转折点**：凯利通过提问引导用户发现自己的构念（如"你似乎在用'成功-失败'这个单一构念看所有事情"）
- **结束状态**：用户意识到自己的构念模式，获得新的视角
- **情绪体验**：从"困惑" → "被点醒" → "原来如此"

##### 验收标准

- [ ] 凯利能在对话中识别用户反复出现的构念模式
- [ ] 凯利以提问而非断言的方式引导用户（苏格拉底式）
- [ ] 用户表示"有启发"或类似反馈
- [ ] 不强行分析，尊重用户的节奏（约束 C1）

##### 约束

- C1：凯利人格一致性，不偏离个人建构心理学框架
- C4：对话内容私密，仅用户可见

##### 依赖

- **需要**：MS-G-01（持续对话基础）
- **启用**：MS-L-06

#### MS-D-03：对话深度不足

**Complex Story：** CS-03
**故事类型：** Dark（从期望到失望）
**优先级：** P2
**故事点数：** 2

##### 用户故事

作为一个**学术研究者（U4）**，
